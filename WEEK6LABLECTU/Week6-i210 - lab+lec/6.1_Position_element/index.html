CSS Positioning Demo

This page demonstrates how different CSS position values work. Scroll through each section carefully and observe how the colored boxes behave.

Static Position (default)

By default, every HTML element is static. This means it appears in the normal order of the page, one after another, like words in a book.

Static Box

Notice how the blue box above sits naturally in the page. The paragraph below flows right under it. Even if we try to give top or left, they will have no effect on a static element.

Relative Position

A relative element starts where it would normally be (just like static), but then we can nudge it using top, left, right, or bottom.

Relative Box

Look at the red box: it moved down and to the right. But notice something: the page still reserves the empty space where the box originally belonged. That’s why the paragraph below does not slide up to replace it.

Relative is like saying: “Start in your normal spot, then shift yourself a bit, but don’t disturb others.”

Absolute Position

An absolute element is completely taken out of the normal page flow. It does not leave empty space behind.

This gray container is the ancestor. An ancestor simply means the outer box that contains another element.

Absolute Box

Because this container has position: relative, the green box is pinned inside it, 20px from the top and 20px from the right.

If no ancestor has positioning set, an absolute element will pin itself to the entire page (body).

Fixed Position

A fixed element is pinned to the browser window (also called the viewport), not to the page.

Fixed Box

The yellow box will always stay at the bottom-right corner of the screen, even if you scroll. Try scrolling down!

This tall block of text forces scrolling. Notice that the fixed yellow box does not move with the rest of the content—it is glued to the screen itself.

Common uses: sticky navbars, “chat now” buttons, cookie banners, or floating “back to top” arrows.

Z-Index (Stacking Order)

When elements overlap, z-index decides which one appears on top. Higher numbers sit above lower numbers.

In this example, green is above blue, and blue is above red, because of their z-index values.

End of demo. Quick summary:
Static = normal flow
Relative = nudge yourself
Absolute = pin to ancestor
Fixed = pin to screen
Z-index = stacking order
